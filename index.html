§ Why FastCUA

The right layer for Windows work.

Not a browser DOM tool. Not vision-only screenshot spam. A resident control plane for desktop apps, system dialogs, and cross-app flows.

Vision-first CUA

  • Limited outside the tab
FastCUA

Desktop + chrome

  • Windows apps & system UI
  • UIA text first; vision on demand
  • Many native acts per turn
  • Pause · approve · interject · exit
§ Principles

Speed from execution design.

Product rules the runtime is built around. Agent step-by-step procedure lives only in the computer-use Skill.

01

Accessibility first

Prefer UIA name / role / value. Screenshots only when pixels add information.

02

One warm control plane

One daemon, one native host, one cursor. Approvals and pause are shared — not rebuilt per click.

03

Many acts per turn

Persistent sky.* JS: click, type, drag, scroll in sequence. Re-observe when layout may change.

04

Apple-style grid

Broken UIA → prefer_vision → grid_view / refine / click_cell. Select ≠ click.

05

Fail fast · 30s

Software work times out in 30s. Human pause and approval waits are intentional blocks.

06

Safe & local

Exact-path whitelist, loopback console, named pipe MCP. Policy stays on the machine.

§ You stay in control

Visible state. Global keys.

Agent tag branching lives in the Skill. Here is what people see and press.

F7

Pause + console

Take over and open the local control center.

F8

Pause / resume

Block or release new desktop actions.

F9

Interject

Pause, type an instruction, auto-resume.

F10

Exit

Shut down FastCUA. Agents must not self-restart.
