FastCUA 为 Computer Use Agent 提供快速、可靠、可控的本地执行层。它把界面信息、原生输入、访问策略与人工接管组织在同一个常驻控制平面中。

01 / PRINCIPLES

速度来自执行方式，
不是宣传数字。

FastCUA 优化 Agent 与计算机之间的往返路径。页面只描述代码已经实现的机制，不展示虚构产品界面。

01Aa

信息优先，视觉按需

界面能够通过可访问性信息准确表达时，直接读取文本与控件结构；画布、自绘控件或视觉验证再请求截图。

02

一个常驻控制平面

客户端共享温热的原生 Host。窗口身份、指针状态、审批、暂停与中断不会为每个动作重复初始化。

03

一次回合，多步执行

稳定的点击、按键、文本输入、滚动与拖拽可以顺序执行；布局或焦点发生变化时，再回到观察与推理。

03 / BOUNDARIES

它是什么，也诚实说明
它暂时不是什么。

FastCUA 是

  • 开源、本地优先的 Computer Use 执行运行时
  • 文本模型与视觉模型都可调用的 MCP 工具层
  • 可暂停、可审批、可人工接管的控制平面
  • 浏览器自动化之外的互补层

当前版本不是

  • 完整的跨平台发布包
  • 替代 DOM、CDP 或所有专用自动化工具
  • 绕过 UAC、安全桌面或认证界面的方案
  • 无需观察变化便能无限批处理的鼠标脚本
