THE FAST PATH

少看重复像素，
多做真实操作。

FastCUA 不强迫 Agent 反复进行“截图 → 推理 → 点击 → 截图”。当界面可以用文本准确表达时，它优先使用 Windows 无障碍树；视觉理解只在真正需要时出现。

01

无障碍树优先，截图按需

按下一步决策需要，独立请求界面文本、截图或两者，减少无用的图像传输与视觉推理。

> include_text: true · include_screenshot: false
02

一个常驻原生 Host

所有客户端共享同一个温热控制平面，窗口身份、指针状态、审批与中断不会在每个动作前重建。

✓
03

一次回合，多步动作

把点击、快捷键、Unicode 输入、滚动、拖拽与连续笔画合并执行，减少反复往返模型。

YOU STAY IN CONTROL

四个全局按键，随时接管。

F7

暂停并打开控制台

立即停止新动作，打开本地控制中心接管电脑。

F8

暂停 / 继续

随时冻结或恢复 Agent 的执行，不打断当前上下文。

F9

展开并插话

展开悬浮岛，向正在执行的任务追加或修正指令。

F10

完全退出

随时关闭 FastCUA，结束本地控制服务。

覆盖浏览器之外的
Windows 桌面。

FastCUA 不替代 CDP 或浏览器自动化；它补上浏览器周围的那一层——应用窗口、系统对话框和跨应用工作流。

01

Windows 桌面应用

操作画图、文件资源管理器、Office 风格应用与其他原生窗口。
