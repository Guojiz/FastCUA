Repeatable self-hosting

From clone to first desktop action in four clear steps.

A local daemon, a compatible native helper, and one MCP entry point. No cloud control plane required.

01 · PREREQUISITES

Prepare Windows

Install Node.js 18+ and place a compatible helper on the machine.

02 · INSTALL

Clone and configure

Set the helper path in config.json or use CUA_BIN for a machine-local override.

03 · RUN

Start the control plane

Run the daemon, then verify the local health endpoint before connecting a client.

Quick start

Designed for copy, paste, verify.

git clone https://github.com/Guojiz/FastCUA.git
cd FastCUA
.\native-host\build.ps1
node daemon.mjs
curl http://127.0.0.1:8420/api/state

Connect an MCP client

Point your client to the thin stdio server. It reuses the shared daemon.

{
  "mcpServers": {
    "sky-computer-use": {
      "command": "node",
      "args": ["C:\\path\\to\\FastCUA\\server.mjs"]
    }
  }
}

For the no-display native host, keep the host binary local and set overlayEnabled to false. See the self-hosting guide for verification and safety notes.
